# 数字员工 - 进度通知规范

## 通知时机

| 阶段 | 通知内容 |
|------|----------|
| PM 完成需求文档 | ✅ 需求文档已生成，请确认 |
| 用户确认后 | 🚀 开始开发... |
| Dev 开始编码 | 💻 Dev Agent 正在编码... |
| Claude Code 启动 | 🤖 Claude Code 开始工作... |
| 编码完成 | ✅ 编码完成，开始测试... |
| QA 开始测试 | 🧪 QA Agent 正在测试... |
| 发现 Bug | ⚠️ 发现 X 个 Bug，正在修复... |
| Bug 修复 | 🔧 修复中... |
| 测试通过 | ✅ 测试全部通过！ |
| 上传 GitHub | 🚀 已上传到 GitHub: [链接] |

## 通知方式

- **Telegram 消息** - 实时通知
- **状态图标** - 🚀 进行中 / ✅ 完成 / ⚠️ 问题 / ❌ 失败

## 示例

```
🚀 数字员工工作流启动

📋 PM Agent - 需求文档已生成
💻 Dev Agent - 正在编码...
🤖 Claude Code - 开始实现登录功能
✅ 编码完成

⚠️ 发现 2 个 Bug
🔧 修复中...
✅ 测试通过
🚀 已上传: https://github.com/Lazily01/xxx
```
