### 5. 🧠 记忆和学习能力

#### 三层记忆架构
```
日志层 (memory/YYYY-MM-DD.md) 
    ↓ 提炼
长期记忆 (MEMORY.md)
    ↓ 提取
工作缓冲 (SESSION-STATE.md)
```

#### WAL Protocol（Write-Ahead Logging）
- **触发：** 更正、偏好、决策、数值
- **流程：** STOP → 写入 SESSION-STATE.md → 再回复

#### 向量搜索
- **工具：** memory_search, memory_get
- **索引：** MEMORY.md + memory/**/*.md
- **技术：** BM25 + 向量 + MMR 重排序

#### 自我进化
- **心跳检查：** 每 30 分钟自动检查系统状态
- **学习循环：** 5 阶段（经历 → 反思 → 概念化 → 实验 → 改进）
- **追踪文件：**
  - `proactive-tracker.md` - 主动行为
  - `recurring-patterns.md` - 重复模式
  - `outcome-journal.md` - 决策跟进

## 🚀 典型工作流

### 1. 项目开发
```
用户："我需要一个音频播放器"
↓
PM Agent 分析需求 → 生成需求文档
↓
用户确认需求
↓
Dev Agent 开发 → QA Agent 测试
↓
GitHub 上线 → 发送链接
```

### 2. AI 生图
```
用户："生成一张 4K 无水印的图，描述：..."
↓
调用豆包 API（doubao-seedream-4-5）
↓
返回图片链接（24 小时有效）
```

### 3. 文章处理
```
用户提供 X 文章链接
↓
twitter-reader 获取内容
↓
总结核心观点
↓
（可选）写入 Obsidian
```

### 4. 网页抓取
```
用户提供 URL
↓
判断是否动态网页
├─ 静态 → web_fetch
└─ 动态 → jina.ai 或 browser
↓
提取内容 → 整理 → 输出
```
