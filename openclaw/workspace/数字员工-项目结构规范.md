# 数字员工 - 项目结构规范

## 标准目录结构

```
项目名称/
├── src/                      # 源代码
│   ├── components/           # 组件
│   ├── pages/                # 页面
│   ├── utils/                # 工具函数
│   └── tests/                # 单元测试
│
├── docs/                     # 📁 文档目录（统一管理）
│   ├── pm/                   # PM Agent 产出
│   │   ├── 需求文档.md
│   │   ├── 技术方案.md
│   │   └── 任务清单.md
│   │
│   ├── qa/                   # QA Agent 产出
│   │   ├── 测试报告.md
│   │   └── bug列表.md
│   │
│   ├── growth/               # Growth Agent 产出
│   │   ├── 增长策略.md
│   │   ├── KPI建议.md
│   │   └── 功能建议.md
│   │
│   └── dev/                  # Dev Agent 产出
│       └── 更新日志.md
│
├── public/                   # 静态资源
├── tests/                    # E2E 测试
│
├── README.md                 # 项目说明
├── package.json              # 依赖配置
└── .gitignore               # Git 忽略

```

## PM Agent 输出规范

```markdown
# 需求文档

## 项目概述
## 功能列表
## 验收标准 ✅  ← 重要！给 QA 用
## 技术要求
```

## QA Agent 输出规范

```markdown
# 测试报告

## 测试概览
## 测试用例
## 测试结果
## Bug 列表
## 建议
```

## Growth Agent 输出规范

```markdown
# 增长策略

## 用户分析
## 增长机会
## 功能建议（优先级 P0/P1/P2）
## KPI 设计
```

## 初始化脚本

```bash
# 创建新项目时
mkdir -p 项目名称/{src/{components,pages,utils,tests},docs/{pm,qa,growth,dev},public,tests}
touch 项目名称/README.md
touch 项目名称/docs/pm/需求文档.md
```

## 注意事项

1. **所有文档放 docs/ 目录** - 不散落在各处
2. **按 Agent 分类** - pm/qa/growth/dev
3. **README 包含链接** - 指向各文档
4. **测试和代码一起提交** - 便于回归
